Speckle

Agent-driven spec building tool — its own plan, rendered as a static prototype of speckle itself.

Prototype. This is a single static HTML file mimicking what speckle serve renders. There's no server: clicking Submit to agent dumps the would-be submission JSON into a banner below the form instead of POSTing it.

1. What speckle is

Speckle is a loop between an AI agent and a human, mediated by a YAML file and a tiny Go server.

  1. The agent writes a .speckle file describing a plan as sections with embedded decisions; each decision has labelled options, optional sandboxed previews, and a place for the user to leave comments.
  2. speckle serve plan.speckle renders the file as HTML and waits.
  3. The user picks options, leaves comments, hits Submit.
  4. The agent reads the submission via speckle await, emits a YAML overlay, applies it with speckle patch. The page hot-reloads and the loop continues.
